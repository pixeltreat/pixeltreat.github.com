HTML head elements

The <head> is a container for all the head section elements. Elements inside <head> can include scripts, instruct the browser where to find style sheets, provide meta information, and more.

The following tags can be added to the head section: <title> <style> <meta> <link> <script> <noscript> and <base>

Within a larger body of text, a piece in italics does not stand out much; instead, it signifies a context difference only while the text is being read.

By contrast, bold face makes text darker than surrounding text. With this technique, the emphasized text strongly stands out from the rest.

Even though it is possible to create nice layouts with HTML tables, tables were designed for presenting tabular data - NOT as a layout tool!

Block Elements

Block level elements normally start (and end) with a new line when displayed in a browser. Ex: <h1> <p> <ul> <table>


Common use of the <div> element, is for document layout. It replaces the "old way" of defining layout using tables.

Inline Elements

Inline elements are normally displayed without starting a new line. Ex: <b> <span> <a> <img>

The HTML <span> element is an inline element that can be used as a container for text.

HTML Forms

HTML forms are used to pass data to a server. An HTML form can contain input elements like text fields checkboxes radio-buttons submit buttons and more. A form can also contain select lists textarea fieldset legend and label elements.

03 HTML 5

Overview

- Of course, we utilize a new DTD, <!DOCTYPE html>

- Other larger features of the spec include canvas, video capabilities, geolocation, local storage, advanced forms,and more.

Older browsers without HTML5 support will usually simply ignore these features with no errors. A handy website to reference is http://caniuse.com, where you can search for HTML5 features and find which browsers support them.
